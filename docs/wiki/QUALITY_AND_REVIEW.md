# ⚖️ Качество и Ручная Проверка (Quality Score & Human Review)

Система OCR-документов автоматически вычисляет "Оценку Качества" (Quality Score) для каждого документа, чтобы решить: можно ли сохранить результат в базу автоматически или требуется вмешательство человека.

Все числовые оценки задаются в диапазоне **[0.0; 1.0]**.

## 1. Компоненты Quality Score

### 1.1 Техническое качество (`technical_quality_score`)
Оценивает качество исходного изображения/PDF-файла:
- Разрешение DPI и наличие низкого разрешения.
- Уровень шума, размытие, сдвиг фокуса.
- Обрезанные края и читабельность текста.

### 1.2 Семантическое качество (`semantic_confidence_score`)
Оценивает успешность работы NLP-модели:
- **Уверенность (Confidence)** по извлеченным полям.
- **Ошибки валидации** (например, ИНН имеет неверную длину).
- Отсутствие обязательных полей (Critical Fields).

### 1.3 Итоговая оценка (`overall_quality_score`)
Взвешенная сумма технического и семантического скоринга. Веса задаются в YAML-профиле (например, 40% техника, 60% семантика).

---

## 2. Маршрутизация (Routing to Human Review)

Для каждого профиля обработки в YAML-конфигурации (например, `court_decision_ru.yaml`) установлены пороги.

**Документ автоматически принимается**, если:
1. `overall_quality_score` >= `overall_quality_min_auto_accept`.
2. Все обязательные поля (`critical_fields`) заполнены.
3. Ошибок валидации нет (`validation_status == "ok"`).

**Отправляется на ручную проверку (Human Review)**, если:
1. `overall_quality_score` ниже порога.
2. Есть критические ошибки (`validation_status == "errors"`).

---

## 3. Human Review API

Для документов, отправленных на ручную проверку, используется отдельный набор контрактов в `api/human_review.py`:
- `GET /human-review/tasks` — получить список документов, ожидающих проверки.
- `POST /human-review/tasks/{id}/submit` — отправить исправленные поля. В результате создается новая версия структурированных данных.

## 4. Логирование и MLflow

Все метрики качества (включая уверенность моделей на каждом поле) логируются в MLflow для дальнейшей аналитики и переобучения NLP-моделей. Это позволяет постоянно улучшать систему и понижать процент ручной проверки.
